El dashboard de Prysm:ID vive en [`app.prysmid.com`](https://app.prysmid.com). Es donde hacés todo lo administrativo del workspace sin código: invitar miembros del equipo, registrar OAuth apps, conectar IdPs sociales, brandear el login, ver audit log, manejar billing.

Cada página tiene su contraparte vía API y vía MCP — esta sección documenta el camino UI. Si preferís código, andá a [/integrate](/integrate/your-saas-app/) o [/agents/tools](/agents/tools/).

## Layout

- **Sidebar izquierdo**: workspaces tuyos. Click en uno para entrar.
- **Tabs por workspace** (Overview, Apps, Users, IdPs, Branding, Billing, Audit, Settings).
- **Top right**: tu avatar + selector de tema (light/dark).

## Lo que NO está en el dashboard (todavía)

- **Crear / editar usuarios end-user** del workspace. Los usuarios se crean por OIDC signup en *tu* app, no acá. Soporte de admin de users vive en la API y MCP por ahora; UI llega en 2026 Q3.
- **SAML SSO config UI**: hoy se hace por API/MCP; UI dedicada en roadmap.
- **Métricas de uso fine-grained**: dashboard tiene contadores básicos; analytics avanzado vía export del audit log.

## Atajos de teclado

| Tecla | Hace |

| `g` `a` | Apps |
| `g` `i` | IdPs |
| `g` `b` | Billing |
| `?` | Mostrar atajos |
